## Why

開發了大量工具散落在電腦各處，隨著時間推移，常忘記工具放在哪裡、做什麼用途。缺乏集中管理的方式，導致重複開發或工具閒置。需要一個中央啟動器面板，讓所有工具的位置、用途、啟動方式一目瞭然，並可直接從面板啟動與監控狀態。

## What Changes

- 建立一個桌面級中央面板應用程式（Tool Launcher Panel），作為所有開發工具的統一入口
- 每個工具透過獨立的 JSON 設定檔註冊，定義啟動路徑、簡稱、說明等資訊
- 面板提供一鍵啟動功能，透過前景 PowerShell 視窗執行目標程式
- 即時狀態燈號：綠色 = 程式運行中，灰色 = 未啟動
- 支援新增、編輯、移除工具設定檔
- 每次開發新工具時，只需建立一個 JSON 設定檔即可納入集中管理

## Capabilities

### New Capabilities

- `tool-registry`: 工具註冊系統 — JSON 設定檔格式定義、工具清單的載入/儲存/掃描機制
- `launcher-engine`: 啟動引擎 — 透過前景 PowerShell 啟動工具程式、處理啟動參數與工作目錄
- `process-monitor`: 程序監控 — 偵測工具是否正在運行、提供綠/灰燈號即時狀態回報
- `central-panel-ui`: 中央面板介面 — 工具清單展示、狀態燈號、一鍵啟動、工具資訊顯示

### Modified Capabilities

（無 — 這是全新專案）

## Impact

- **新增檔案結構**: 整個專案為全新建立，包含應用程式原始碼、工具設定檔目錄、專案配置
- **技術選型**: 需決定 UI 框架（桌面應用）、程序管理方式
- **外部依賴**: PowerShell（Windows 內建）、可能的 UI 框架依賴
- **使用流程改變**: 開發新工具後，建立 JSON 設定檔即可在面板中管理與啟動
